# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly.

### Contact

Send an email to: [hyperpsgit@proton.me](mailto:hyperpsgit@proton.me)

### Guidelines

* Provide a clear and detailed description of the issue.
* Include steps to reproduce the vulnerability.
* Attach proof-of-concept code or screenshots if applicable.
* Specify the affected version and environment.

### Response Time

* Initial acknowledgment within 48 hours.
* Status updates will be provided as the investigation progresses.
* Critical issues will be prioritized and addressed as soon as possible.

### Disclosure Policy

* Do not publicly disclose the vulnerability until it has been reviewed and resolved.
* Once fixed, coordinated disclosure may be made with proper credit to the reporter.

### Scope

This policy applies to all components of the repository, including:

* Backend (Laravel, Core PHP)
* API endpoints
* Database interactions
* Authentication and session management
* Voting logic and IP restriction mechanisms

### Out of Scope

* Issues caused by misconfiguration of local environments
* Non-security related bugs or feature requests
* Theoretical issues without a valid exploit path

Thank you for helping improve the security of this project.
